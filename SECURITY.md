# Security Policy

## Reporting a vulnerability

If you discover a security issue, please report it privately.

Preferred reporting options:

- Open a private security issue on GitHub
- Use GitHub's security advisory tools if available

Please include:

- A description of the issue
- Steps to reproduce
- Expected versus actual behavior
- Any relevant files or runtime context

## Response

Security issues are reviewed as soon as possible. Fixes will be released in a timely manner and documented in `CHANGELOG.md` when appropriate.
