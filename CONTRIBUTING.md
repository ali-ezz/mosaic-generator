# Contributing to Mosaic Generator

Thank you for contributing to this project. Your improvements help make the mosaic tool more useful and reliable.

## How to contribute

1. Open an issue to start a discussion for bug fixes, feature requests, or enhancements.
2. Create a branch with a clear name:
   - `feature/<short-description>` for new functionality
   - `bugfix/<short-description>` for bug fixes
3. Make your changes in the branch and keep commits focused.
4. Submit a pull request against `main`.

## Pull request expectations

- Include a short summary of what changed and why.
- Reference any related issue number.
- Add simple reproduction steps or examples when applicable.
- Keep changes small and reviewable.

## Branch naming

Use one of these patterns:
- `feature/<brief-description>`
- `bugfix/<brief-description>`
- `docs/<brief-description>`

## Review process

- Pull requests should be approved before merging.
- Reviewers may ask for updates or clarifications.
- Contributions should follow the existing project style and documentation.

## Reporting issues

- Use the bug report template for bugs.
- Use the feature request template for new ideas.
- Add as much detail as possible so maintainers can reproduce and evaluate the request.

## Additional notes

This repository follows the `CODE_OF_CONDUCT.md`. Please read it before contributing.
